## 1. 设计原则

### 1.1 核心原则

| 原则 | 描述 | 实现要求 |
|------|------|----------|
| **直觉导航** | 重要功能2次点击内可达 | 功能分层，核心操作≤2次点击 |
| **节奏流畅** | 高频操作响应≤0.1s | 面板动画≤0.15s |
| **层级清晰** | 状态机无歧义 | 面板打开时锁定场景切换 |
| **反馈明确** | 每次操作有视觉/音效反馈 | 按钮点击、伤害数值、状态变化 |
| **防御性设计** | 数据操作有校验和容错 | 空数据保护、边界检查 |
| **知情决策** | Roguelike核心原则 | 战斗前提供敌人/奖励预览 |

### 1.2 设计语言

**视觉风格**：东方玄幻美学 + 现代游戏UI清晰度

- 深邃星空背景，呼应"星陨"主题
- 紫金配色（紫色为主+暗金点缀），传达修仙文化的神秘感
- 发光效果与光晕，模拟灵力流动
- 几何线条与留白，兼顾信息密度与呼吸感
- 暗金色(#D4AF37)作为装饰边框，强化东方古典质感

### 1.3 字体规范

| 用途 | 字体 | 字号 | 颜色 |
|------|------|------|------|
| 标题 | 方正经 / Noto Sans SC Bold | 24px | #FFD700 |
| 副标题 | Noto Sans SC Semibold | 18px | #E0D0FF |
| 正文 | Noto Sans SC Regular | 14px | #CCCCCC |
| 数值 | JetBrains Mono / Roboto Mono | 16px | #FFD700 |
| 数值增加 | - | 14px | #4CAF50 |
| 数值减少 | - | 14px | #F44336 |

### 1.4 分辨率适配

| 分辨率 | 基准缩放 | 最小字号 |
|--------|----------|----------|
| 1920×1080 | 1.0x | 14px |
| 2560×1440 | 1.0x | 14px |
| 3840×2160 | 1.5x | 18px |

**适配规则**：
- 使用Godot锚点/容器布局，避免固定像素
- 关键元素（技能按钮、节点卡片）设置最小/最大尺寸限制
- 支持21:9超宽屏，敌人区域可水平滚动

## 3. 游戏状态机

### 3.1 分层架构设计

采用**主状态 + 面板状态属性**的分层架构，避免状态数指数增长：

```gdscript
# 主状态枚举
enum MainState {
    MAIN_MENU,
    CHARACTER_SELECT,
    HUB,
    MAP,
    BATTLE,
    VICTORY,
    DEFEAT,
    PAUSED,
    SETTINGS
}

# 面板类型枚举
enum PanelType {
    NONE,
    EQUIPMENT,
    INVENTORY,
    SHOP,
    FORGE,
    QUEST,
    ACHIEVEMENT,
    FACTION,
    CHARACTER_STATUS,
    SKILL_CONFIG,
    BATTLE_PREVIEW,
    WORDLY_INSIGHT  # 词缀汇总
}

# 状态管理器
var main_state: MainState = MainState.MAIN_MENU
var is_panel_open: bool = false
var current_panel: PanelType = PanelType.NONE
var previous_main_state: MainState  # 用于ESC返回
```

### 3.2 状态定义

| 主状态 | 场景文件 | 触发条件 |
|--------|----------|----------|
| `MAIN_MENU` | main.tscn | 游戏启动 |
| `CHARACTER_SELECT` | character_select.tscn | 新游戏/继续 |
| `HUB` | hub_scene.tscn | 角色选择后/返回 |
| `MAP` | map_scene.tscn | 点击地图按钮 |
| `BATTLE` | battle_scene.tscn | 战前预览确认后 |
| `BATTLE_PREVIEW` | battle_preview_panel.tscn | 点击地图节点 |
| `VICTORY` | victory_panel.tscn | 战斗胜利 |
| `DEFEAT` | defeat_panel.tscn | 战斗失败 |
| `PAUSED` | pause_panel.tscn | ESC暂停 |
| `SETTINGS` | settings_panel.tscn | 暂停菜单内 |

### 3.4 状态转换规则

| 当前状态 | 触发 | 下一状态 | 条件 |
|----------|------|----------|------|
| 任意 | ESC | PAUSED | 全局中断 |
| PAUSED | ESC/继续 | 上一主状态 | - |
| HUB | M | MAP | - |
| MAP | ESC/返回 | HUB | - |
| MAP | 点击节点 | BATTLE_PREVIEW | - |
| BATTLE_PREVIEW | 确认 | BATTLE | - |
| BATTLE_PREVIEW | 取消/ESC | MAP | - |
| BATTLE | ESC | PAUSED | - |
| BATTLE | 胜利 | VICTORY | 敌人HP≤0 |
| BATTLE | 失败 | DEFEAT | 玩家HP≤0 |
| VICTORY/DEFEAT | 返回 | MAP | - |
| HUB/MAP (面板打开) | ESC | 关闭面板或暂停 | 取决于面板类型 |

### 3.5 面板锁定规则

```
规则：面板打开时禁止场景切换
例外：ESC始终可以打开暂停菜单

面板状态属性：
- is_panel_open: bool
- current_panel: PanelType
- previous_main_state: MainState（用于ESC返回）
```

### 3.6 异常状态处理

| 异常情况 | 处理方式 |
|----------|----------|
| 战斗中退出游戏 | 保存当前进度，标记run为"未完成" |
| 存档失败 | 回退到上一个有效存档，显示错误提示 |
| 战斗异常 | 标记节点为"未完成"，返还挑战次数 |
| 网络异常（云存档） | 降级到本地存档，回滚操作 |

## 附录A：状态管理代码示例

```gdscript
# game.gd 状态管理（分层架构）

enum MainState {
    MAIN_MENU,
    CHARACTER_SELECT,
    HUB,
    MAP,
    BATTLE,
    VICTORY,
    DEFEAT,
    PAUSED,
    SETTINGS
}

enum PanelType {
    NONE,
    EQUIPMENT,
    INVENTORY,
    SHOP,
    FORGE,
    QUEST,
    ACHIEVEMENT,
    FACTION,
    CHARACTER_STATUS,
    SKILL_CONFIG,
    BATTLE_PREVIEW,
    WORDLY_INSIGHT
}

var main_state: MainState = MainState.MAIN_MENU
var previous_main_state: MainState
var is_panel_open: bool = false
var current_panel: PanelType = PanelType.NONE

func _input(event: InputEvent) -> void:
    if event is InputEventKey and event.pressed:
        match event.keycode:
            KEY_ESCAPE:
                _handle_escape()
            KEY_M:
                if can_open_map(): _show_map()
            KEY_E:
                if can_open_equipment(): _show_equipment_panel()
            KEY_I:
                if can_open_inventory(): _show_inventory_panel()
            KEY_C:
                if can_open_character(): _show_character_panel()
            KEY_RIGHT:
                if is_controller_input(): _focus_next()
            KEY_LEFT:
                if is_controller_input(): _focus_prev()

func _handle_escape() -> void:
    match main_state:
        MainState.PAUSED:
            _resume_from_pause()
        MainState.HUB, MainState.MAP, MainState.BATTLE:
            if is_panel_open:
                _close_current_panel()
            else:
                _show_pause_menu()
        MainState.BATTLE_PREVIEW:
            _cancel_battle_preview()
        _:
            if is_panel_open:
                _close_current_panel()

func open_panel(panel_type: PanelType) -> void:
    if not can_switch_scene():
        return
    previous_main_state = main_state
    is_panel_open = true
    current_panel = panel_type
    # 实例化面板...

func close_panel() -> void:
    is_panel_open = false
    current_panel = PanelType.NONE
    # 关闭面板...

func can_switch_scene() -> bool:
    return not is_panel_open

func _show_map() -> void:
    if not can_switch_scene():
        return
    main_state = MainState.MAP
    # 切换场景...

func _show_battle_preview(node_data: MapNode) -> void:
    previous_main_state = main_state
    main_state = MainState.BATTLE_PREVIEW
    current_panel = PanelType.BATTLE_PREVIEW
    # 显示预览面板...

func confirm_battle() -> void:
    main_state = MainState.BATTLE
    # 进入战斗...
```
