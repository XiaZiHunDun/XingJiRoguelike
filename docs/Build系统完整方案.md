# Build系统完整方案
> 装备、词缀、随机性、技能搭配 - 最终推荐方案

---

## 一、核心设计理念

**参考成功游戏的设计哲学：**

| 游戏 | Build核心驱动力 | 我们的借鉴 |
|------|---------------|-----------|
| Slay the Spire | 卡牌组合产生的化学反应 | 元素反应作为Build桥梁 |
| Hades | 恩盖组合产生独特Build | 词缀+套装产生独特Build |
| Hades | 武器变体决定基础风格 | 武器元素决定技能池 |
| 暗黑破坏神 | 词缀随机性+套装效果 | 装备词缀+套装系统 |

**我们的Build核心循环：**
```
选取装备 → 激活词缀效果 → 触发元素反应 → 享受Build红利 → 掉落更好装备
     ↑                                                        ↓
     └──────────────── 循环验证 ────────────────────────────────┘
```

**三条Build路线：**
1. **元素爆发流** - 火/冰/雷直伤 + 元素反应加成
2. **ATB控制流** - ATB加速/倒退/冻结 + 回合控制
3. **防御反击流** - 高防/护盾 + 反弹/反击

## 二、装备随机性设计

### 2.1 随机时机：掉落时生成

```
装备生成时机 = 战斗掉落时
装备等级 = 随当前进度（章节1-10）
词缀数量 = 稀有度决定
词缀数值 = 范围内浮动
```

**理由：** 预生成会降低刷装动力，掉落时生成保证每场战斗都有期待感

### 2.2 装备等级系统

| 装等 | 名称 | 基础属性倍率 | 词缀数值范围 | 来源 |
|------|------|-------------|-------------|------|
| T1 | 凡品 | 1.0x | 100% | 章节1-3 |
| T2 | 精品 | 1.2x | 90%-110% | 章节2-5 |
| T3 | 极品 | 1.5x | 85%-115% | 章节4-7 |
| T4 | 史诗 | 1.8x | 80%-120% | 章节6-9 |
| T5 | 传说 | 2.2x | 75%-125% | 章节8-10 |
| T6 | 神话 | 2.5x | 70%-130% | BOSS掉落 |

**词缀数值公式：**
```
实际数值 = 基础数值 × 装等倍率 × (random范围)
例如：火伤+15% 的 T3极品装 → 15% × 1.5 × (0.85~1.15) = 19.1%~25.9%
```

### 2.3 词缀数量由稀有度决定

| 稀有度 | 词缀数量 | 套牢词缀 | 随机词缀 | 特殊词缀 |
|--------|---------|---------|---------|---------|
| 凡品(白) | 1 | 0 | 1 | 0 |
| 精品(绿) | 1-2 | 0 | 1-2 | 0 |
| 极品(蓝) | 2 | 1 | 1 | 0 |
| 史诗(紫) | 2-3 | 1 | 1-2 | 0 |
| 传说(橙) | 3 | 1 | 2 | 1(强力) |
| 神话(红) | 4 | 1 | 2 | 1(极强) |

**"锁定词缀"机制：**
- 蓝装开始有1个锁定词缀（装备自带的定义词缀，如武器的基础词缀）
- 掉落的词缀数量是在锁定词缀之外的随机

### 2.4 词缀重锻系统

**每个村庄/商店提供1次重锻机会：**

| 操作 | 消耗 | 效果 |
|------|------|------|
| 重随其他词缀 | 30金币 | 将1个随机词缀换成同槽位池中的另一个 |
| 锁定词缀 | 50金币 | 锁定1个词缀不被重随 |
| 升级词缀 | 100金币 | 将词缀数值+10%（上限130%） |

**重锻不是核心机制，是"补救机制" - 避免玩家因为刷到垃圾词缀而卡关**

## 三、词缀效果系统实现

### 3.1 词缀分类重构

将原来的6类重组为4类，更利于代码实现：

| 新分类 | 原分类 | 效果时机 | 实现方式 |
|--------|--------|---------|---------|
| **伤害加成** | 伤害类+元素反应类 | 伤害计算时 | `final_damage *= (1 + value)` |
| **ATB调节** | ATB类 | ATB变化时 | `atb_speed += value` |
| **生存能力** | 防御/生存类+特殊类 | 受伤计算时 | `damage_taken *= (1 - value)` |
| **能量调节** | 能量类 | 能量操作时 | `max_energy += value` |

### 3.2 词缀效果应用流程

```
技能使用 → BattleManager.计算伤害()
           ↓
    AffixSystem.计算伤害加成()
           ↓
    遍历所有激活的词缀
    ├── fire_damage_15: if skill.element == FIRE → damage *= 1.15
    ├── crit_damage_25: → crit_multiplier += 0.25
    └── lifesteal_5: → lifesteal += 0.05
           ↓
    返回 [final_damage, crit_multiplier, lifesteal, ...]
```

### 3.3 具体词缀效果代码

```gdscript
# systems/equipment/affix_system.gd

class_name AffixSystem

# 激活的词缀列表
var active_affixes: Array[Dictionary] = []

func apply_affix_effect(affix_id: StringName, entity, context: Dictionary):
    """应用词缀效果"""
    var affix = DataManager.get_affix(affix_id)
    if affix.is_empty():
        return

    match affix.type:
        Enums.AffixType.DAMAGE:
            _apply_damage_affix(affix, entity, context)
        Enums.AffixType.ATB:
            _apply_atb_affix(affix, entity, context)
        Enums.AffixType.ENERGY:
            _apply_energy_affix(affix, entity, context)
        Enums.AffixType.SPECIAL:
            _apply_special_affix(affix, entity, context)

func _apply_damage_affix(affix: Dictionary, entity, context: Dictionary):
    var value = affix.value
    match affix.id:
        &"fire_damage_15":
            if context.get("element") == Enums.Element.FIRE:
                context.damage_multiplier *= (1.0 + value)
        &"ice_damage_15":
            if context.get("element") == Enums.Element.ICE:
                context.damage_multiplier *= (1.0 + value)
        &"thunder_damage_15":
            if context.get("element") == Enums.Element.THUNDER:
                context.damage_multiplier *= (1.0 + value)
        &"phys_damage_10":
            if context.get("element") == Enums.Element.PHYSICAL:
                context.damage_multiplier *= (1.0 + value)
        &"crit_damage_25":
            context.crit_multiplier += value
        &"lifesteal_5":
            context.lifesteal += value
        # 元素反应类词缀
        &"evaporation_boost":
            context.reaction_boost[Enums.ElementReaction.EVAPORATION] = value
        &"superconduct_boost":
            context.reaction_boost[Enums.ElementReaction.SUPERCONDUCT] = value
        # ... 其他词缀

func _apply_atb_affix(affix: Dictionary, entity, context: Dictionary):
    match affix.id:
        &"atb_speed_10":
            entity.atb_component.bonus_speed += affix.value * entity.atb_component.base_speed
        &"atb_speed_15":
            entity.atb_component.bonus_speed += affix.value * entity.atb_component.base_speed
        &"atb_drain_10":
            context.atb_drain_on_hit += affix.value
        &"atb_drain_20":
            context.atb_drain_on_hit += affix.value
        &"atb_freeze":
            context.atb_freeze_on_hit += affix.value
        &"perfect_timing":
            context.perfect_timing_bonus += affix.value
```

## 四、技能来源分配

### 4.1 当前问题分析

**问题：武器给3个技能太多**

| 当前设计 | 问题 |
|---------|------|
| 武器3技能 | Build选择太少，只有3选3 |
| 饰品0技能 | 饰品没有Build价值 |
| 护甲2技能 | 防御技能没有独特性 |

### 4.2 推荐方案：1+1+1+1结构

```
每件装备 = 1个核心技能（固定）
           +
         X个辅助技能（可选）

武器  =  1核心技能（元素攻击）+ 1-2辅助技能（可选）
护甲  =  1核心技能（防御技能）+ 1辅助技能（可选）
饰品  =  1核心技能（强力特殊）+ 0辅助
灵石  =  0技能 + 被动效果
```

### 4.4 技能联动设计

**元素反应联动（核心Build路径）：**

```
火技能 + 冰技能 = 蒸发反应（额外伤害）
    +
火魂戒(烈焰爆发) = 蒸发伤害×1.5
    +
蒸发强化词缀 = 再次×1.5
    =
Combo: 蒸发反应基础伤害 × 2.25
```

**ATB控制联动：**

```
冰技能(减速) + 雷技能(ATB倒退) + 疾风戒(时间膨胀)
    =
敌人ATB几乎不动，我方ATB飞速
```

## 五、灵石系统

### 5.1 灵石定位

**灵石 = 元素亲和强化，不是技能来源**

| 槽位 | 解锁方式 |
|------|---------|
| 灵石槽1 | 默认解锁 |
| 灵石槽2 | 章节3解锁 |
| 灵石槽3 | 章节6解锁 |

### 5.2 灵石随机性

**灵石掉落时随机生成：**

| 属性 | 随机范围 |
|------|---------|
| 元素类型 | 火/冰/雷 (33%各) |
| 等级 | 下级(70%) / 上级(30%) |
| 数值浮动 | 90%-110% |

### 5.3 灵石合成（局外成长）

```
3个下级灵石 = 1个随机上级灵石
```

**合成公式：** 提供3个选择，让玩家选择想要的结果

## 六、套装设计

### 6.1 当前问题

- 只有3套，数量太少
- 4件套对于7个装备槽来说太难凑

### 6.2 推荐方案：5套3-4件混搭

| 套装 | 件数 | 路线 | 2件效果 | 3件效果 |
|------|------|------|---------|---------|
| 雷劫 | 4 | 修真 | 雷伤+25% | 雷系攻击15%概率重置ATB |
| 烈焰之心 | 3 | 修真 | 火伤+20% | 灼烧可叠加2层 |
| 量子丹田 | 4 | 科技 | 能量恢复+30% | 释放技能时ATB+15% |
| 磐石 | 3 | 中立 | 防御+20% | 受到伤害15%概率免疫 |
| 闪电之心 | 3 | 科技 | 雷伤+15% | ATB清零概率+10% |

**混搭规则：**
- 同一路线2件 = 小奖励
- 同一路线3件 = 中等奖励
- 同一路线4件 = 强大奖励
- **跨路线混搭 = 只有基础属性，无套装效果**

### 6.3 套装效果强度

**原则：套装效果应该是"锦上添花"而非"必须"**

- 散件Build应该也能通关
- 套装效果让Build更顺畅
- 不需要刻意刷套装

## 七、完整Build示例

### 7.1 Build示例1：火系爆发流

```
装备选择：
- 武器：火焰剑（火焰斩、火球术、灼烧之刃）
- 护甲：法袍（魔力护盾、恢复）
- 饰品：火魂戒（烈焰爆发）+ 暴击戒（致命一击）
- 灵石：火灵石(上)、火灵石(下)、雷灵石(下)

激活词缀：
- fire_damage_15（火伤+15%）
- crit_damage_25（暴击伤害+25%）
- crit_rate_10（暴击率+10%）
- lifesteal_5（吸血5%）
- perfect_timing（完美时机+10%）

战斗流程：
1. 火焰斩起手 → 附加灼烧 + 火伤词缀加成
2. 火球术跟进 → 火焰+冰（敌人）= 蒸发反应（高伤害）
3. 暴击戒激活 → 致命一击
4. 灼烧之刃收割 → 完美时机 + 火伤词缀 = 爆炸伤害

预期效果：
- 蒸发反应：基础伤害 × 2.0（蒸发）× 1.15（火伤）= 2.3倍
- 暴击时：再 × 1.25（暴击伤害）+ 1.15（完美时机）= 约3倍
```

装备选择：

- 敌人每次行动前被我方打掉20-30% ATB
- 我方ATB速度比敌人快50%+
- 敌人几乎无法行动
```

### 7.3 Build示例3：防御反击流

```
装备选择：
- 武器：精钢剑（横斩、突刺、重击）

## 九、总结

| 系统 | 设计要点 |
|------|---------|
| 装备随机性 | 掉落时生成、装等决定倍率、词缀范围浮动 |
| 词缀系统 | 4大类30个词缀、统一AffixSystem处理 |
| 技能分配 | 武器2-3、护甲1-2、饰品1、灵石0 |
| 灵石系统 | 元素强化、3槽位、局外合成 |
| 套装系统 | 5套3-4件、同一路线凑效果 |

**Build核心循环：**
```
元素技能 → 触发反应 → 享受加成 → 获得更强装备 → 强化Build
```

这套方案的关键是**让每件装备都有独特价值，让词缀组合产生1+1>2的效果**。
