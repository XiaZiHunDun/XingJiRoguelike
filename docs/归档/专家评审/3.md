#### ⚠️ 发现的不一致/潜在问题

**1. 区域等级设计 - 关键问题**

文档显示地图5仍为**统一57-70级**（沙海回声1级起始→地图5为57-70级），这与专家评审建议的"区域起始等级+27"方案不符：

```
当前实现（文档8.2）：
沙海回声(1级) → 地图5: 57-70级 ❌
专家评审建议：
沙海回声(1级) → 地图5: 28级 ✅
```

**影响：** 玩家通关第一个区域后直接面对满级BOSS，难度曲线断裂，其他区域前4张地图失去意义。

**2. 词缀数量不一致**

- 设计文档：41个词缀
- 实现文档：45个词缀（`affix_data.gd`）
- 具体差异：魔法增强型从5个扩展到9个，可能包含重复或新增词缀

**建议：** 统一词缀数量，避免平衡失控。

**3. ATB软上限调整**

| 版本 | 软上限 | 50级溢出伤害 |
|------|--------|--------------|
| 原设计 | 200 | +327% |
| 修订版 | **250** | **+277%** |
| 建议值 | 300 | +227% |

当前250的软上限仍导致**277%的溢出伤害**，配合速度共鸣（ATB>300时+30%溢出伤害），实际可达**307%**。这个数字仍然过高，建议：

```
方案A：软上限提升至300（溢出227%，+共鸣257%）
方案B：溢出倍率从1%降至0.5%（250上限时溢出138%）
方案C：两者结合（300上限+0.5%倍率=113%）
```

```
✓ 词缀系统通过tags支持新共鸣类型
✓ 区域系统模板化，新增区域只需配置数据
```

#### ⚠️ 潜在风险点

**1. 循环依赖风险**

```gdscript
# systems/affixes/resonance_system.gd 依赖
# entities/player/player.gd 依赖
# systems/affixes/affix_effects.gd

# 建议：使用事件总线或接口解耦
```

**2. 数值计算顺序问题**

`player.gd`中的属性计算链：

```gdscript
base_attributes → equipment_bonuses → amplifier_multipliers → effective_attributes
```

**风险：** 如果`equipment_bonuses`包含来自词缀的加成，而词缀效果又依赖`effective_attributes`，可能形成循环。

**建议：** 明确计算阶段：
```gdscript
# 阶段1：基础属性（等级+永久增幅）
# 阶段2：装备基础属性（不含词缀）
# 阶段3：词缀效果（基于阶段2）
# 阶段4：增幅器乘法（最终加成）
```

**3. 存档系统未明确**

`run_state.gd`管理全局状态，但未看到存档/加载实现。Phase 1完成后应补充：
- 角色永久增幅库存
- 已解锁传送门
- 阵营关系（Phase 3）

#### ⚠️ 需要调整的数值

**1. 速度溢出伤害过高**

```
50级基础敏捷142.5 → ATB速度527
软上限250 → 溢出277点 → +277%伤害

问题：配合装备词缀轻松达到ATB>300
结果：速度Build伤害远超其他Build
```

**建议调整：**
```gdscript
# core/consts.gd
const ATB_SOFT_CAP: float = 300.0  # 从250提升至300
const ATB_OVERFLOW_MULTIPLIER: float = 0.005  # 从0.01降至0.5%
# 结果：50级溢出113%，配合共鸣143%，仍强但不过度
```

**2. 敌人HP与玩家伤害匹配**

根据数值框架文档：

| 敌人 | HP | 玩家伤害(装备后) | 击杀下数 | 时长 |
|------|-----|------------------|----------|------|
| 普通怪(50级) | 1050 | 850×1.2(词缀)=1020 | 2-3下 | 4-6秒 |
| 精英怪(50级) | 3150 | 1020 | 4-5下 | 8-12秒 |
| BOSS(50级) | 10500 | 1020 | 11-12下 | 22-26秒 |

**结论：** 实际战斗时长可能**短于**目标（BOSS 30-40秒），但这是好事，符合Roguelike快节奏。如需延长BOSS战，可提高BOSS防御或增加阶段机制。
