# 星陨纪元 UI设计规范 v3.0 - 专家评审报告

---

## 总体评价

这是一份**专业且完整度极高**的UI设计规范文档，结构清晰、细节到位，充分体现了"修仙Roguelike"的品类理解。设计方向与Godot技术栈高度匹配，具备直接进入开发的成熟度。

---

## 优点

- **色彩系统专业且可扩展**：星空紫金配色准确传达修仙神秘感，GDScript常量定义方式便于开发实现，节点色/元素色/状态色的分层设计非常清晰。

- **状态机设计严谨**：12个状态的转换规则明确，面板锁定规则解决了"面板内误触场景切换"的常见问题，`previous_state`的设计考虑了ESC返回的连贯性。

- **战斗界面信息架构成熟**：ATB时间轴 + 敌我状态分区 + 技能按钮的布局是经过验证的成熟方案，能量消耗显示、冷却倒计时、完美时机等细节体现了对ATB系统的深度理解。

- **装备对比与词缀汇总设计出色**：装备对比的"绿色+/红色-"色差规则、词缀汇总面板的元素加成汇总，是Roguelike构筑玩法的核心体验支撑点。

- **防御性设计意识贯穿全文**：空数据保护、边界检查、悬停Tooltip延迟、确认对话框等细节，体现了对玩家误操作和边缘情况的充分考虑。

## 问题与建议

### 🔴 严重问题（影响核心体验）

**1. 枢纽界面缺乏"角色展示"与"快速编队"入口**

**问题描述**：
枢纽界面布局中，核心功能仅包含地图/装备/背包/设置四个按钮，但修仙Roguelike的核心是**角色成长可视化**和**Build构筑**。当前设计中，玩家无法在枢纽直接看到：
- 当前角色的立绘/模型
- 核心属性面板（攻击/防御/暴击/灵力等）
- 技能配置/切换入口

玩家每次想查看角色状态都需要打开装备面板，这与"直觉导航"原则相悖。

**建议方案**：
在枢纽界面中央区域（地图按钮上方或左侧）增加**角色展示区**：
```
┌─────────────────────────────────────────────────────┐
│                                                     │
│          [角色立绘/3D模型 - 150×200px]              │
│          ┌─────────────────────────┐                │
│          │ 攻击: 45   暴击: 12%    │                │
│          │ 防御: 30   灵力: 25     │                │
│          │ [技能配置] [属性详情]    │                │
│          └─────────────────────────┘                │
│                                                     │
│           ┌─────────────────┐                       │
│           │     地 图        │                       │
│           │     [M]          │                       │
│           └─────────────────┘                       │
└─────────────────────────────────────────────────────┘
```
或者在次要功能行增加"技能"按钮，点击后打开技能配置面板。

---

**2. 地图节点交互规则存在歧义**

**问题描述**：
5.5节规定"已解锁-普通/精英 → 点击直接进入战斗"，但Roguelike玩家在节点选择时通常需要**权衡决策信息**（如：可能掉落什么？敌人类型是什么？）。直接进入战斗剥夺了玩家的战术准备时间。

同时，5.6节只设计了BOSS确认对话框，普通/精英节点没有确认环节，这与"战前预览"的设计意图矛盾（6.6节设计了战前预览面板，但普通/精英点击后是否展示预览未明确）。

**建议方案**：
统一节点点击行为：
```
┌─────────────────────────────────────┐
│           ⚔ 普通战斗                │
├─────────────────────────────────────┤
│  敌人预览: 沙海斥候 ×2, 沙蝎 ×1     │
│  可能掉落: 玄铁(60%), 星尘(100%)    │
│  难度: ★★☆☆☆                      │
│                                     │
│     [ 取消 ]        [ 挑战 ]         │
└─────────────────────────────────────┘
```
所有节点（除已完成/锁定）点击后均弹出预览面板，BOSS节点增加难度警告标记。这符合Roguelike"知情决策"的品类惯例。

---

**3. 状态机缺少"战前预览"状态**

**问题描述**：
状态机定义中，从MAP到BATTLE的转换是"点击节点 → BATTLE"，但6.6节设计了战前预览面板。这意味着存在一个**中间态**（预览面板打开），但状态机未定义该状态。

当预览面板打开时：
- 是否可以ESC关闭？（应该可以）
- 是否可以切换到其他界面？（不应该）
- 此时游戏暂停还是继续？（应该暂停）

**建议方案**：
增加状态 `BATTLE_PREVIEW`：
```
MAP → (点击节点) → BATTLE_PREVIEW → (确认) → BATTLE
                     ↓
                   (取消) → MAP
```
预览面板打开时，遵循"面板打开时禁止场景切换"规则，ESC关闭预览返回MAP。

### 🟡 重要优化（提升体验）

**1. 战斗界面ATB时间轴需要"预输入"提示**

**问题描述**：
ATB战斗的核心爽点在于**连续行动的节奏感**。当前设计中，时间轴仅显示"即将行动→第二位→第三位"，但玩家无法预判"我这次行动后，下次轮到我是什么时候"。

这导致玩家无法做出"用低费技能还是攒能量放大招"的策略决策。

**建议方案**：
在ATB时间轴中增加**下一轮预测**：
```
ATB行动顺序:
[当前回合]        [下一回合预测]
[敌B●]→[敌A]→[玩家]  →  [敌C]→[敌B]→[?]→[玩家]
                         ↑ 用虚线/半透明表示
```
或增加一个**ATB速度对比指示器**：
```
你的速度: 120  |  敌人平均速度: 95
下一轮行动预计: 2.3秒后
```

---

**2. 装备对比系统缺少"快速替换"功能**

**问题描述**：
7.3节装备面板中，每个装备槽都有[卸下]按钮和对比信息，但**缺少一键替换**功能。玩家需要：卸下当前装备 → 找到背包中目标装备 → 装备，三步操作。

在Roguelike高频装备更替的场景下，操作成本偏高。

**建议方案**：
增加**双击替换**逻辑：
- 背包中双击装备：自动对比并替换对应槽位
- 或增加[替换]按钮在对比面板中：
```
┌─────────────────────────────────────┐
│ 武器对比                            │
│ 当前: 星际之剑 Lv.3 (攻击+15)       │
│ 候选: 沙漠之刃 Lv.4 (攻击+18)       │
│                                     │
│ [取消]  [卸下当前]  [装备候选]       │
└─────────────────────────────────────┘
```

---

**3. 失败结算缺少"复活"机制设计**

**问题描述**：
7.10节失败结算仅提供[重新开始]和[返回主菜单]两个选项。作为Roguelike游戏，永久死亡是核心，但**有限复活机会**（如使用稀有道具、消耗记忆碎片）是提升留存的重要手段。

当前设计未体现任何"继续游戏"的可能性，可能导致玩家挫败感过强。

**建议方案**：
增加复活选项（条件性显示）：
```
┌─────────────────────────────────────┐
│           战 斗 失 败                │
├─────────────────────────────────────┤
│                                     │
│  你倒在了沙海中...                  │
│                                     │
│  ┌─────────────────────────────┐   │
│  │ 💫 使用「记忆碎片×3」复活    │   │
│  │    复活后恢复50%生命         │   │
│  └─────────────────────────────┘   │
│                                     │
│  [ 重 新 开 始 ]    [ 返回 主 菜 单 ]│
└─────────────────────────────────────┘
```
记忆碎片是底部资源栏已有资源，可作为付费复活货币。

---

**4. 枢纽界面的次要功能行信息密度不足**

**问题描述**：
次要功能行包含6个按钮（商店/锻造/采集/任务/成就/阵营），但：
- 采集功能在Roguelike中通常是节点类型，枢纽内是否需要独立入口？
- 任务和成就功能相近，可合并？
- 阵营声望系统是否为核心功能？若非核心，可移至次级菜单

当前设计可能导致功能入口冗余，违反"直觉导航"原则。

**建议方案**：
将次要功能重组为：
```
┌─────────────────────────────────────────────────────────────┐
│  [商店]  [锻造]  [任务]  [图鉴]  [设置]                     │
│   🛒      🔨      📜      📖      ⚙                         │
└─────────────────────────────────────────────────────────────┘
```
- 采集：移至地图节点类型（已有）
- 成就：合并至任务面板的"成就"标签页
- 阵营：若非核心，移至图鉴或任务内作为子功能

---

**5. 快捷键体系缺少"取消当前操作"的通用键**

**问题描述**：
9.1节全局快捷键中，ESC功能过载：
- 暂停/关闭面板
- 但在某些状态下ESC有不同行为（如战斗内Space是暂停，ESC是打开暂停菜单）

玩家容易混淆"我想取消当前选择"和"我想打开菜单"。

**建议方案**：
引入**右键取消**作为通用操作：
- 背包内选中物品后，右键取消选中
- 技能瞄准时，右键取消施法
- 面板内，右键等同于ESC关闭面板

这是PC游戏的事实标准，符合Steam玩家习惯。

### 🟢 建议改进（锦上添花）

**1. 增加"灵力/能量预消耗"视觉反馈**

战斗中，悬停在技能按钮上时，可高亮显示ATB时间轴上"释放该技能后的新顺序"：
```
悬停[技能2⚡5]时：
ATB时间轴显示: [玩家] → (释放技能后延迟2.0s) → [敌A]→[玩家]
```
增强策略决策的可视化。

---

**2. 词缀汇总面板增加"词缀冲突"提示**

某些词缀可能存在互斥（如火系加成与冰系加成无法同时生效），在汇总面板中用⚠标记提示玩家。

---

**3. 商店增加"回购"标签页**

防止玩家误售重要装备，增加最近出售的5件物品回购功能。

---

**4. 战斗界面的"逃跑"按钮增加成功率显示**

```
[逃跑 85%]
```
悬停显示："基础成功率60% + 速度差加成25%"

---

**5. 动画规范补充"加载过渡动画"**

全屏切换时，0.3s淡出→淡入可能产生"闪黑"感，建议增加一个**星云流转**的Loading动画（0.5s循环），避免玩家感知到卡顿。

## 开放式问题

1. **ATB战斗的"完美时机"机制设计深度**
   文档提到"完美时机时子弹时间(0.2x) + 特写"，这是一个很好的操作感增强设计。但需要明确：
   - 完美时机的判定窗口是多少？（建议50-100ms）
   - 是否所有技能都有完美时机？还是仅限特定技能？
   - 完美时机的奖励是什么？（伤害加成？额外回合？）

2. **Roguelike的"局外成长"系统位置**
   文档提到了记忆碎片、时砂等资源，但未明确它们的局外用途。建议在枢纽增加一个"传承"或"天赋"入口，用于消耗局外资源解锁永久加成，这是Roguelike留存的核心。

3. **Steam Deck / 手柄适配考量**
   当前快捷键体系完全是键鼠逻辑，但Steam平台有大量手柄用户。建议预留手柄输入接口，至少支持：
   - 十字键/摇杆导航
   - A确认/B取消
   - LB/RB切换标签页

4. **性能基准测试标准**
   动画规范很详细，但需要明确性能目标：
   - 最低配置下的帧率要求？（建议i5-4590 + GTX 960 保持60fps）
   - 粒子效果的最大数量限制？
   - UI重绘的优化策略？（如血条变化使用Tween而非每帧set）

---

## 评审总结

本UI设计规范达到了**可直接进入开发**的成熟度，核心架构设计合理，细节考虑周全。主要改进方向集中在：

1. **强化角色构筑的可视化**（枢纽角色展示区）
2. **统一节点交互逻辑**（预览面板化）
3. **补全状态机中间态**（BATTLE_PREVIEW）
4. **增强战斗决策信息量**（ATB预测）

建议开发团队优先完成P0/P1项目后再启动全面UI实现，并在原型阶段快速验证枢纽布局和战斗ATB时间轴的可用性。
