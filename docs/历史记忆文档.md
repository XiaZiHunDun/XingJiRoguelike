### 4.2 Godot 4 兼容性问题（重要经验）

#### 问题1：静态工厂方法
**问题描述**: Godot 4 不支持在 `static func` 中调用 `ClassName.new()`

**受影响的代码**:
```gdscript
# 错误示例 - 会导致编译错误
static func create_fireball() -> CardData:
    return CardData.new()  # Error!
```

**解决方案**: 移除所有静态工厂方法，改用直接实例化

#### 问题2：枚举引用格式
**问题描述**: 嵌套枚举需要通过 `class_name` 类型访问

**正确格式**:
```gdscript
const _Enums = preload("res://scripts/enums.gd")
match card_type:
    _Enums.CardType.ATTACK: pass
```

#### 问题3：TypedArray声明
**问题描述**: `Array[ClassName]` 语法在某些情况下导致类型推断错误

**解决方案**:
```gdscript
# 错误
var talents: Array[TalentData]

# 正确
var talents: Array
```

#### 问题4：预加载依赖
**问题描述**: 嵌套类无法直接访问外部类的预加载常量

**解决方案**:
```gdscript
class EnemyAgent:
    const _UnitData = preload("res://scripts/unit_data.gd")
```

## 七、技术总结与最佳实践

### 7.1 Godot 4 开发注意事项

1. **`class_name` 必须在文件顶部声明**
2. **`static func` 中不能使用 `ClassName.new()`**
3. **嵌套枚举访问需要通过 `class_name` 类型前缀**
4. **预加载常量必须在类体顶部声明**
5. **TypedArray 语法在某些情况下会导致推断错误**
6. **内部类需要单独预加载依赖**

### 7.2 推荐的代码模式

```gdscript
# ==================== 推荐模式 ====================

# 1. 类型声明与预加载
class_name MyClass
extends Node

const _Enums = preload("res://scripts/enums.gd")
const _External = preload("res://external.gd")

# 2. 实例化（普通方法内）
func create_instance():
    var instance = _External.new()
    return instance

# 3. 枚举使用
func get_card_name(type: int) -> String:
    match type:
        _Enums.CardType.ATTACK: return "攻击"
        _Enums.CardType.DEFENSE: return "防御"

# ==================== 避免模式 ====================

# 1. 静态方法中实例化 - 会报错
static func create() -> MyClass:
    return MyClass.new()  # Error!

# 2. 直接使用未声明的枚举 - 会报错
match type:
    CardType.ATTACK: pass  # Error! 需要 CardRarity.ATTACK

# 3. TypedArray声明 - 可能报错
var items: Array[ItemData]  # Error!
```

### 7.3 Autoload加载顺序

```
Types (类型注册) → BattleManager (战斗) → GameState (状态) → AudioManager (音频)
```

## 九、经验教训

### 9.1 成功经验

1. **AI辅助开发效率高** - 快速生成代码框架和修复Bug
2. **模块化设计** - 便于独立测试和调试
3. **行为树AI** - 扩展性好，易于添加新敌人类型
4. **ATB战斗系统** - 融合回合制与即时战斗的优点

### 9.2 需要改进

1. **预加载策略** - 应该一开始就规划好依赖关系
2. **静态方法使用** - 了解Godot 4的限制后再使用static
3. **测试策略** - 应该更早使用Godot编辑器进行实际测试
4. **代码审查** - 多次修复导致代码结构混乱

### 9.3 新项目建议

1. 先设计完整的数据结构再写代码
2. 建立代码规范，特别是预加载和类型声明
3. 使用Godot编辑器开发，避免headless模式验证困难
4. 频繁与用户确认需求，避免返工
