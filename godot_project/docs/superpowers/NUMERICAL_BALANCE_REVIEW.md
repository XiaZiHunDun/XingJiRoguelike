## 一、ATB战斗系统数值分析

### 1.1 软上限300设计评估

| 参数 | 数值 | 评估 |
|------|------|------|
| ATB软上限 | 300 | 合理，从250提高到300增加容错空间 |
| 速度软上限 | 200 | 需配合ATB软上限调整 |
| 动能上限 | 30% | 合理，动能转化为伤害加成上限 |

**问题发现**:
- `consts.gd` 中存在两个软上限定义: `ATB_SOFT_CAP = 300.0` 和 `SPEED_SOFT_CAP = 200.0`
- 速度软上限200与ATB软上限300的关联计算未在代码中明确体现
- 建议: 确认速度200对应ATB 300的换算关系是否正确

### 1.2 溢出伤害计算

```
公式: 溢出伤害加成 = (ATB值 - 300) * 0.005 (0.5%/点)
示例: ATB = 350 时, 溢出伤害加成 = 50 * 0.005 = 25%
```

**问题发现**:
- 注释与实际计算存在微小差异: 代码注释写"0.5% per point"但常量定义为0.005(已正确)
- `battle_calculator.gd` 第48-51行正确实现了此公式

### 1.3 时砂系统经济模型

| 参数 | 数值 | 评价 |
|------|------|------|
| 最大时砂 | 2次 | 偏少，建议3次 |
| 恢复条件 | 每击杀5敌 | 合理 |
| 子弹时间倍率 | 0.2x | 过于强力，建议0.5x |
| 暂停持续时间 | 3.0秒 | 合理 |

**问题发现**:
- 时砂初始2次对于高难度战斗可能不足
- 子弹时间0.2x导致战斗节奏过慢

## 二、境界突破数值分析

### 2.1 境界属性加成表

| 境界 | 等级范围 | 格子数 | 突破消耗 | 属性要求(体质/精神/敏捷) | 特权 |
|------|----------|--------|----------|--------------------------|------|
| 凡人身 | 1-10 | 1 | 50 | 25/20/20 | 基础修炼 |
| 感应境 | 11-20 | 1 | 100 | 45/35/35 | 星尘感应 |
| 聚尘境 | 21-30 | 2 | 150 | 70/55/55 | 能量凝聚 |
| 凝核境 | 31-40 | 3 | 200 | 100/80/80 | 核爆之力 |
| 星尘境 | 41-50 | 3 | 300 | 140/110/110 | 星尘之躯 |
| 粒子境 | 51-60 | 3 | 300 | 200/160/160 | 粒子化 |
| 星火境 | 61-70 | 3 | 0 | 0/0/0 | 终极形态 |

**问题发现**:
- **境界要求跳跃过大**: 从星尘境到粒子境，体质从140→200(+43%)，消耗却相同(300)
- **粒子境到星火境突破消耗为0**: 最高境界无代价突破，设计不合理
- **星尘境/粒子境突破消耗相同(300)**: 但属性要求增加约43%，性价比不一致

### 2.2 属性成长计算

```
HP = 100 + 体质 * 8
Energy = 5 + 精神 * 0.5
ATB速度 = 100 + 敏捷 * 3
```

**评价**: 基础公式简洁，但境界突破时的属性断层需要关注

|--------|--------------|------|
| 白(WHITE) | 1-1 | 偏少 |
| 绿(GREEN) | 1-2 | 合理 |

| 紫(PURPLE) | 2-3 | 合理 |

## 四、资源经济分析

### 4.1 星尘获取/消耗

**获取渠道**:
- 守墓人敌人击杀: 20-40星尘/次
- 战斗胜利: 待确认

**消耗渠道**:
- 境界突破: 50-300星尘/次
- 商店购买: 待确认

**问题发现**:
- 星尘获取量(20-40)与突破消耗(50-300)差距过大
- 初期突破需击杀2-3波敌人，中期需10-15波
- 建议增加基础星尘获取量或降低突破消耗

**评价**: 极品质性价比是初品质的5倍，但价格为100记忆碎片，需确认获取难度

## 六、关键数值问题汇总

### 严重问题

1. **时砂初始2次不足**: 高难度战斗需要更多容错
2. **粒子境突破消耗与星尘境相同但属性要求高43%**: 性价比不一致
3. **星火境突破消耗为0**: 最高境界无代价，设计缺失

### 中等问题

1. **白色装备词缀数量与绿色重叠**
2. **红色装备攻击力未能体现品质差距**
3. **疾风套装4件效果描述模糊**
4. **星尘获取与突破消耗比例失衡**

### 轻微问题

1. **子弹时间0.2x过强**
2. **区域等级重叠度高**

## 七、平衡改进建议

### 7.1 ATB系统

```
建议调整:
- 时砂最大次数: 2 → 3
- 子弹时间倍率: 0.2x → 0.5x
- 或增加时砂恢复条件: 5敌 → 4敌
```

### 7.2 境界系统

```
建议调整:
- 星尘境→粒子境: 突破消耗300 → 250 (性价比平衡)
- 星火境突破: 增加最终挑战环节，非零代价
- 或星火境突破消耗200星尘，保留经济循环
```

### 7.3 装备系统

```
建议调整:
- 白色装备: 词缀数量0-1
- 红色武器基础攻击: 60-80 (当前55)
- 明确疾风套装4件效果的数值
```

### 7.4 经济系统

```
建议调整:
- 守墓人星尘奖励: 20-40 → 30-60
- 或降低突破消耗: 全局*0.8
```

## 八、附录: 关键代码引用

### A. ATB软上限计算 (battle_calculator.gd:26-32)
```gdscript
static func calculate_speed_with_soft_cap(base_speed: float, bonus_speed: float = 0.0) -> Array:
    var total = base_speed + bonus_speed
    if total > Consts.SPEED_SOFT_CAP:
        var excess = total - Consts.SPEED_SOFT_CAP
        var kinetic = minf(excess * Consts.SPEED_BONUS_TO_KINETIC, Consts.KINETIC_ENERGY_CAP)
        return [Consts.SPEED_SOFT_CAP, kinetic]
    return [total, 0.0]
```

### B. 溢出伤害计算 (battle_calculator.gd:48-51)
```gdscript
static func calculate_overflow_bonus(overflow_amount: float) -> float:
    if overflow_amount <= 0:
        return 0.0
    return overflow_amount * Consts.ATB_OVERFLOW_MULTIPLIER  # 0.5% per point
```

### C. 时砂恢复逻辑 (battle_calculator.gd:180-184)
```gdscript
static func calculate_time_sand_restore(current_kills: int) -> Array:
    var kills = current_kills + 1
    if kills >= 5:
        return [true, 0]  # 恢复时砂，重置计数
    return [false, kills]
```

**评审结论**: 游戏数值框架设计合理，ATB系统创新性高。核心问题集中在境界突破性价比、时砂初始次数和经济循环平衡。建议优先修复星火境突破设计缺失和粒子境性价比问题。
