## 二、测试框架选型

### 2.1 推荐：GUT Framework

**理由：**
- Godot 4官方推荐的测试框架
- 支持单元测试和集成测试
- 社区活跃，维护良好
- 易于集成到CI/CD

**安装方式：**
```bash
# 通过Godot AssetLib安装
# 或手动下载: https://github.com/bitwes/Gut

# 目录结构
godot_project/
├── tests/
│   ├── test_battle_calculator.gd  # 伪测试文件（已创建）
│   ├── run_tests.gd               # 测试运行器
│   └── unit/
│       ├── test_atb.gd
│       ├── test_energy.gd
│       └── test_damage.gd
```

### 2.2 替代方案：godot-autosim

**理由：**
- 支持自动化模拟测试
- 适合复杂场景测试
- 但社区较小

## 七、已知限制

1. **Godot环境依赖**：部分测试需要Godot运行时
2. **随机性**：伤害计算包含randf()，需要固定种子
3. **UI耦合**：部分组件与UI紧耦合，难以独立测试
4. **外部依赖**：存档系统依赖文件系统
