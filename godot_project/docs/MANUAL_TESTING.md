# UI 面板人工测试文档

> 测试日期：2026-04-17
> 测试范围：Phase 1-4 已实现的 UI 面板
> 测试目的：验证 UI 交互、数据显示、状态同步

---

## 测试环境准备

### 启动游戏
1. 打开 Godot 4.6.2
2. 加载项目 `godot_project`
3. 点击运行（或按 F5）
4. 从主菜单选择角色（星际战士/奥术师）

### 进入据点
游戏开始后会进入据点（Hub），显示角色信息和菜单按钮。

---

## 测试用例

### 1. 商店面板

**入口**：据点菜单 → 商店

**测试步骤**：
1. 点击「商店」按钮
2. 观察星尘余额显示是否正确
3. 观察商品列表是否显示（应有6个随机材料）
4. 点击任意商品的「购买」按钮
5. 观察星尘是否扣除
6. 观察库存是否减少
7. 点击「关闭」按钮

**预期结果**：
- [ ] 星尘余额正确显示
- [ ] 商品列表显示材料名称、库存、价格
- [ ] 购买后星尘扣除
- [ ] 购买后库存-1
- [ ] 关闭后返回据点

**边界情况**：
- [ ] 星尘为0时尝试购买（应提示"星尘不足"）
- [ ] 库存为0时购买按钮应禁用或提示"已售罄"

### 2. 背包面板

**入口**：据点菜单 → 背包

**测试步骤**：
1. 点击「背包」按钮
2. 观察默认显示「装备」标签页
3. 切换到「材料」标签页
4. 切换到「消耗品」标签页
5. 点击物品的「详情」或「使用」按钮（消耗品）
6. 点击「关闭」按钮

**预期结果**：
- [ ] 装备标签页显示背包装备列表
- [ ] 材料标签页显示所有拥有材料及数量
- [ ] 消耗品标签页显示消耗品及使用按钮
- [ ] 点击使用后效果生效（如回血）
- [ ] 关闭后返回据点

9. 点击「关闭」按钮

**预期结果**：
- [ ] 锻造系统正常初始化
- [ ] 装备详情显示正确
- [ ] 锁定词缀功能正常
- [ ] 锻造消耗显示正确
- [ ] 锻造成功后装备更新
- [ ] 关闭后返回据点

**预期结果**：
- [ ] 主线/支线任务分类显示
- [ ] 任务状态正确显示（未解锁/可接取/进行中/已完成/已领取）
- [ ] 接取任务后状态更新
- [ ] 领取奖励后获得星尘/记忆碎片
- [ ] 关闭后返回据点

### 11. 境界面板

**入口**：据点中点击境界显示（或菜单中有入口）

**测试步骤**：
1. 观察当前境界名称
2. 观察等级进度条
3. 观察突破需求（体质/精神/敏捷）
4. 观察突破消耗星尘数量
5. 观察特殊能力和增幅器格子

**预期结果**：
- [ ] 境界名称正确
- [ ] 等级进度条显示当前/最大
- [ ] 突破需求显示当前值/需求值（满足时绿色✓，不满足时红色✗）
- [ ] 达到最大境界时显示"已达最高境界"

### 12. 叙事弹窗（势力剧情）

**触发条件**：加入势力、达到声望里程碑、完成关键任务

**测试步骤**：
1. 执行上述触发条件之一
2. 观察是否出现叙事弹窗
3. 观察淡入动画
4. 等待"继续"按钮可点击
5. 点击"继续"
6. 观察淡出动画

**预期结果**：
- [ ] 弹窗淡入显示
- [ ] 标题和内容正确显示
- [ ] 淡入动画完成后按钮可点击
- [ ] 点击后弹窗淡出并关闭
