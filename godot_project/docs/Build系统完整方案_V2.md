# Build系统完整方案 V2
> 基于专家评审 + 设计迭代的最终方案

---

## 一、核心设计理念

### 设计哲学

```
【传统Roguelike】
拿到什么用什么 → 被动接受随机性

【我们的方案】
看到随机性 → 做决策 → 构建Build → 享受化学反应
```

### 参考游戏

| 游戏 | Build核心 | 我们的借鉴 |
|------|----------|-----------|
| Hades | 武器变体+恩盖组合 | 词缀触发联动 |
| Slay the Spire | 卡牌化学反应 | 元素反应机制化 |
| 暗黑破坏神 | 词缀随机+套装 | 高随机性Build |

### 三条Build路线

1. **元素爆发流** - 火/冰/雷 + 元素反应加成
2. **ATB控制流** - 速度/倒退/冻结 + 回合控制
3. **防御反击流** - 高防/护盾 + 反弹/反击

### Phase策略

- **Phase 1**：火/冰/雷三角，核心机制验证
- **Phase 2+**：风/土系，根据反馈扩展

## 二、角色技能系统（少而精）

### 设计原则

```
- 4个核心技能，提供基本战斗手段
- 玩家开局就有完整Identity
- 不依赖掉落，保证基础游戏体验
- 装备是Build主体，不是"给技能"
```

## 三、装备系统（高随机性）

### 3.1 核心设计：取消稀有度/装等

```
【取消】
- 稀有度系统（凡品/稀有/史诗/传说）
- 装等系统（T1-T6）

【替代】穿戴要求系统
- 装备掉落时随机生成穿戴要求
- 玩家属性/技能/等级满足要求才能穿戴
- 高要求 = 通常意味着高收益（但不是绝对）
```

### 3.2 装备掉落随机性

```
每件装备掉落时，完全随机生成：

【技能】
├─ 数量：0-4个（概率分布）
│   ├─ 0技能：15%（纯词缀装）
│   ├─ 1技能：30%
│   ├─ 2技能：30%
│   ├─ 3技能：20%
│   └─ 4技能：5%（稀有）
│
├─ 类型：从装备类型对应的技能池抽取
│   例：武器技能池（15个技能）→ 抽取2-3个
│
└─ 等级：1-3级（影响伤害/效果）

【词缀】
├─ 数量：1-4个
├─ 类型：从40+词缀池随机
└─ 数值：基础值 ± 10-20%

【穿戴要求】（三维随机组合）
├─ 属性要求（5种）
├─ 技能要求（3种）
└─ 等级要求（2种）
```

### 3.3 穿戴要求系统

```
【属性要求】（5种，随机1-3种组合）
├─ 基础攻击 >= X（10/15/20/25/30/35...）
├─ 防御 >= X（5/10/15/20/25...）
├─ 最大HP >= X（50/80/100/120/150...）
├─ ATB速度 >= X%（5%/10%/15%/20%/25%...）
└─ 能量上限 >= X（3/4/5/6...）

【技能要求】（3种，随机1-2种组合）
├─ 元素技能数量 >= X（1/2/3）
├─ 终结技等级 >= X（1/2/3）
└─ 某元素类型（必须有火/冰/雷系技能）

【等级要求】（2种，随机1种）
├─ 玩家等级 >= X（5/10/15/20/25...）
└─ 完成章节数 >= X（1/2/3/4...）
```

## 五、词缀联动系统

### 5.1 联动类型

```
【装备↔角色联动】
- 装备技能命中 → 角色终结技伤害+X%
- 装备技能击杀 → 角色技能冷却-X秒
- 角色使用技能 → 装备技能效果+X%

【装备↔装备联动】
- 火系技能 + 冰系技能命中 → 触发蒸发反应
- 护盾存在 + 装备技能 → 伤害+X%
- 两件装备词缀同时满足条件 → 触发额外效果

【词缀↔词缀联动】
- 同时拥有"炽焰之触"和"灼烧蔓延" → 灼烧可传染
- 同时拥有"暴击回能"和"能量涌动" → 暴击恢复更多能量
```

### 5.2 联动示例

```
【火系Build联动链】

装备：火焰剑
├─ 技能：烈焰斩、灼烧爆发
├─ 词缀：火伤+20%、炽焰之触、蒸发引擎
└─ 触发：暴击附加灼烧

装备：火魂戒
├─ 技能：烈焰爆发
├─ 词缀：火伤+15%、灼烧蔓延
└─ 触发：灼烧可引爆扩散

角色：终结技（流星）
└─ 被火系词缀加成

【实际战斗】
1. 烈焰斩命中 → 附加灼烧（火伤+20%加成）
2. 暴击触发 → 炽焰之触 → 额外灼烧
3. 敌人已有灼烧 + 冰系技能命中 → 触发蒸发
4. 蒸发引擎 → 蒸发保留火元素
5. 灼烧叠满5层 → 火凤涅盘 → 引爆扩散
6. 引爆伤害享受所有火系加成
```

## 七、灵石系统

### 7.1 灵石定位

```
- 0主动技能
- 被动效果 + 元素赋予
- 元素赋予 = 改变技能属性（核心功能）
```

### 7.2 元素赋予

```
【基础赋予】
├─ 火灵石：技能转化为火属性
├─ 冰灵石：技能转化为冰属性
└─ 雷灵石：技能转化为雷属性

【赋予示例】
玩家用物理武器"横斩"
镶嵌火灵石后 → 变成"火焰横斩"（火属性）

玩家用角色"流星"（可选择元素）
镶嵌冰灵石后 → 变成"寒冰流星"（冰属性）
```

### 7.3 灵石共振（未来扩展）

```
三同元素（3个火灵石）：元素主宰 - 灼烧可无限叠加
三不同元素（各1个）：元素协调 - 所有反应伤害+30%
两同一异（2火1雷）：元素侵蚀 - 攻击时有机率施加第三种DOT
```

## 十二、总结

**Build核心循环：**
```
选择角色（定Identity）
    ↓
战斗掉落（惊喜盒）
    ↓
检查穿戴要求（能否穿上？）
    ↓
决策（换/不换/保留）
    ↓
词缀联动（化学反应）
    ↓
成长（属性/等级提升）
    ↓
解锁更多装备
```

**这套方案的关键设计：**

1. **角色有基底** - 4个固定技能，保证基本体验
2. **装备是主体** - 高随机性，玩家主要精力所在
3. **穿戴要求** - 三维随机，没有"垃圾装等"
4. **词缀联动** - 触发型40%，产生化学反应
5. **代价词缀** - 高风险高回报，增加张力
6. **标签共鸣** - 散件也有价值，不必须凑套
7. **元素赋予** - 灵石改变技能属性，灵活多变
